## 二、开发约定

| 约定 | 说明 |
|------|------|
| 包名规范 | `com.junsong.{服务名}`，如 `com.junsong.contract` |
| 实体命名 | 模块前缀 + 业务名，如 `ConContract`（con = contract 缩写） |
| 端口分配 | 业务服务用 92xx / 93xx，避免与现有端口冲突（见部署手册服务清单） |
| **配置位置** | 路由、数据源、Redis、白名单等业务配置**全部放 Nacos**，本地仅保留连接信息 |
| **禁止** | 禁止 Java 硬编码路由；禁止本地配置文件写数据源密码 |
| 接口鉴权 | 控制器方法用 `@RequiresPermissions("模块:业务:操作")` 声明权限 |
| 统一返回 | 列表用 `TableDataInfo`，单条/操作用 `AjaxResult`，继承 `BaseController` |
| 操作日志 | 增删改方法加 `@Log(title=..., businessType=...)` |
| 数据源选择 | 需动态数据源/数据权限用 `junsong-common-datasource`；独立简单数据源参考 gen/job |

**Mapper 接口** `mapper/ConContractMapper.java`：

```java
package com.junsong.contract.mapper;

import java.util.List;
import com.junsong.contract.domain.ConContract;

public interface ConContractMapper {
    ConContract selectConContractByContractId(Long contractId);
    List<ConContract> selectConContractList(ConContract contract);
    int insertConContract(ConContract contract);
    int updateConContract(ConContract contract);
    int deleteConContractByContractIds(Long[] contractIds);
}
```

**Mapper XML** `resources/mapper/contract/ConContractMapper.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.junsong.contract.mapper.ConContractMapper">
    <resultMap type="ConContract" id="ConContractResult">
        <id     property="contractId"   column="contract_id"/>
        <result property="contractNo"   column="contract_no"/>
        <result property="contractName" column="contract_name"/>
        <result property="amount"       column="amount"/>
        <result property="createBy"     column="create_by"/>
        <result property="createTime"   column="create_time"/>
    </resultMap>

    <select id="selectConContractList" parameterType="ConContract" resultMap="ConContractResult">
        select contract_id, contract_no, contract_name, amount, create_by, create_time
        from con_contract
        <where>
            <if test="contractName != null and contractName != ''">
                and contract_name like concat('%', #{contractName}, '%')
            </if>
        </where>
        order by contract_id desc
    </select>

    <select id="selectConContractByContractId" parameterType="Long" resultMap="ConContractResult">
        select contract_id, contract_no, contract_name, amount from con_contract
        where contract_id = #{contractId}
    </select>

    <insert id="insertConContract" parameterType="ConContract" useGeneratedKeys="true" keyProperty="contractId">
        insert into con_contract (contract_no, contract_name, amount, create_by, create_time)
        values (#{contractNo}, #{contractName}, #{amount}, #{createBy}, sysdate())
    </insert>

    <update id="updateConContract" parameterType="ConContract">
        update con_contract
        <set>
            <if test="contractName != null">contract_name = #{contractName},</if>
            <if test="amount != null">amount = #{amount},</if>
        </set>
        where contract_id = #{contractId}
    </update>

    <delete id="deleteConContractByContractIds" parameterType="String">
        delete from con_contract where contract_id in
        <foreach item="id" collection="array" open="(" separator="," close=")">#{id}</foreach>
    </delete>
</mapper>
```

**Controller** `controller/ConContractController.java`：

```java
package com.junsong.contract.controller;

import java.util.List;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import com.junsong.common.core.web.controller.BaseController;
import com.junsong.common.core.web.domain.AjaxResult;
import com.junsong.common.core.web.page.TableDataInfo;
import com.junsong.common.core.utils.poi.ExcelUtil;
import com.junsong.common.log.annotation.Log;
import com.junsong.common.log.enums.BusinessType;
import com.junsong.common.security.annotation.RequiresPermissions;
import com.junsong.contract.domain.ConContract;
import com.junsong.contract.service.IConContractService;

/**
 * 合同管理 Controller
 *
 * @author Genesis·峻松
 */
@RestController
@RequestMapping("/contract")
public class ConContractController extends BaseController {

    @Autowired
    private IConContractService conContractService;

    @RequiresPermissions("contract:contract:list")
    @GetMapping("/list")
    public TableDataInfo list(ConContract contract) {
        startPage();
        List<ConContract> list = conContractService.selectConContractList(contract);
        return getDataTable(list);
    }

    @RequiresPermissions("contract:contract:query")
    @GetMapping("/{contractId}")
    public AjaxResult getInfo(@PathVariable("contractId") Long contractId) {
        return success(conContractService.selectConContractByContractId(contractId));
    }

    @RequiresPermissions("contract:contract:add")
    @Log(title = "合同", businessType = BusinessType.INSERT)
    @PostMapping
    public AjaxResult add(@RequestBody ConContract contract) {
        return toAjax(conContractService.insertConContract(contract));
    }

    @RequiresPermissions("contract:contract:edit")
    @Log(title = "合同", businessType = BusinessType.UPDATE)
    @PutMapping
    public AjaxResult edit(@RequestBody ConContract contract) {
        return toAjax(conContractService.updateConContract(contract));
    }

    @RequiresPermissions("contract:contract:remove")
    @Log(title = "合同", businessType = BusinessType.DELETE)
    @DeleteMapping("/{contractIds}")
    public AjaxResult remove(@PathVariable Long[] contractIds) {
        return toAjax(conContractService.deleteConContractByContractIds(contractIds));
    }
}
```

> 注意控制器的 `@RequestMapping("/contract")` 是**服务内部路径**，网关会通过 `StripPrefix=1` 去掉路由前缀后转发。

### 步骤 8：网关路由与鉴权白名单

在 Nacos 编辑 `junsong-gateway-dev.yml`，新增路由：

```yaml
spring:
  cloud:
    gateway:
      server:
        webflux:
          routes:
            # ……已有路由……
            - id: junsong-contract
              uri: lb://junsong-contract
              predicates:
                - Path=/contract/**
              filters:
                - StripPrefix=1
```

> - `uri: lb://junsong-contract` 中的名称必须与 `bootstrap.yml` 的 `spring.application.name` 一致。
> - `StripPrefix=1` 表示转发时去掉第一段路径（`/contract`），控制器内 `@RequestMapping("/contract")` 与之配合，最终前端访问 `/prod-api/contract/contract/list`。
> - **必须使用新前缀** `spring.cloud.gateway.server.webflux.routes`（Spring Cloud 2025.x），旧前缀 `spring.cloud.gateway.routes` 不生效。

合同接口需要登录鉴权，**无需**加入白名单。仅当某接口要匿名访问时，才在 `security.ignore.whites` 中添加。

修改后重启 Nacos 与网关：

```bash
docker compose -f docker/docker-compose.yml restart junsong-nacos
sleep 45
docker restart junsong-gateway
```

## 四、服务间调用（Feign）

当新服务需要调用其他服务（如查用户信息）时，使用 Feign。参考 `junsong-api/junsong-api-system`：

1. 在 `junsong-api` 下定义远程接口（`@FeignClient(contextId="...", value="junsong-system")`）和 DTO。
2. 在调用方 POM 引入对应的 `junsong-api-xxx` 依赖。
3. 启动类已有 `@EnableRyFeignClients`，直接 `@Autowired` 注入远程接口即可调用。

> 调用必须携带内部请求头令牌，框架的 Feign 拦截器会自动透传当前用户上下文。

## 五、在已有服务中新增接口（最常见）

如果只是给现有服务（如 finance）加一个新功能，**无需新建服务**，步骤大幅简化：

1. 在对应模块写 `domain` / `mapper` / `service` / `controller`（同步骤 5）。
2. 写建表 SQL + 菜单权限 SQL（同步骤 6）。
3. **无需**改网关路由（该服务路由已存在）。
4. **无需**改 Nacos 配置（除非用到新的配置项）。
5. 重新编译该服务并更新：
   ```bash
   mvn clean package -DskipTests -pl junsong-modules/junsong-finance -am
   docker cp junsong-modules/junsong-finance/target/junsong-modules-finance.jar \
     junsong-modules-finance:/home/junsong/junsong-modules-finance.jar
   docker restart junsong-modules-finance
   ```
6. 前端加 API 与页面，重新 build 前端。
