## 报告漏洞

如果你发现了安全漏洞，请**不要**通过公开的 Issue 或 PR 提交，以免在修复前暴露给恶意利用者。

### 报告方式

1. **首选**：通过 GitHub 私有安全公告提交
   - 前往仓库主页 → Security → Advisories → New draft security advisory
   - 或访问：https://github.com/siruscs/JunSong/security/advisories/new

2. **备选**：发送加密邮件至维护者
   - 请在邮件标题注明 `[SECURITY]`
   - 描述漏洞影响范围、复现步骤、建议修复方案（如有）

### 报告内容建议包含

- 漏洞类型（如：SQL 注入、XSS、权限绕过、敏感信息泄露等）
- 受影响版本
- 详细复现步骤（最小可复现环境）
- 概念验证代码（PoC，可选）
- 建议的修复方案（可选）

---

## 披露政策

我们遵循**负责任披露**原则：

1. 收到报告后，维护者会在 **48 小时内**确认收到
2. 评估漏洞严重程度和影响范围
3. 在合理时间内开发并验证补丁
4. 补丁发布前**不公开**漏洞细节
5. 补丁发布后，公开致谢报告者（如报告者同意）

## 已知安全注意事项

### 当前已实现的安全措施

- **密码存储**：用户密码使用 BCrypt 哈希存储，盐值随机生成
- **JWT 令牌**：使用 RS256 非对称加密签名，Token 有效期可控
- **接口鉴权**：基于 RBAC 的权限控制，接口粒度到按钮级别
- **SQL 注入防护**：MyBatis 参数化查询，禁止字符串拼接 SQL
- **XSS 防护**：前端 Vue 模板自动转义，后端输出过滤
- **敏感信息脱敏**：配置文件中的密码、密钥使用占位符，不提交真实值
- **请求限流**：Gateway 层基于 Sentinel 实现接口限流与熔断

### 部署时需注意的安全事项

- **生产环境必须修改默认密码**：Nacos（nacos/nacos）、MySQL root、Redis
- **关闭调试模式**：生产环境关闭 Swagger、Actuator 敏感端点
- **启用 HTTPS**：公网部署必须配置 SSL/TLS
- **安全组/防火墙**：仅开放必要端口（80/443/22），其余端口不对公网开放
- **定期更新依赖**：关注 Spring Boot、Vue、MySQL 等依赖的安全公告

## 安全相关配置检查清单

部署生产环境前，请逐项确认：

- [ ] 所有默认密码已修改为强密码
- [ ] `application.yml` / `bootstrap.yml` 中无硬编码敏感信息
- [ ] Nacos 配置中的密码已替换为真实值（非占位符）
- [ ] Swagger / Knife4j 已关闭或限制访问（生产环境）
- [ ] Actuator 敏感端点已禁用或受保护
- [ ] 服务器安全组仅开放必要端口
- [ ] 已启用 HTTPS（公网环境）
- [ ] 日志中不打印密码、Token、身份证号等敏感信息
- [ ] 数据库连接使用 SSL（跨网络部署时）
- [ ] 定期备份数据库和 Nacos 配置
